Journal des changements à JCAssertion

Version 1.0.9

  • Ajout du type d'assertion CompteFichiers
  • Correction d'un bogue de mumérotation des cas dans le journal.
  • Correction du bogue qui ne permettait pas plus d'une connectionOracle dans un essai. La connection à oracle restait toujours sur le compte défini par le premier ConnectionOracle
  • Ajout du paramètre /MAX au programme JCAssertion.exe. Ce paramètre spécifie iun nnombre maximal d'assertion qui peuvent échouer avant que le programme cesse d'évaluer les assertions

Version 1.0.8

  • Ajout du type d'assertion AssertXPath
  • Pour ExporteLOB lorsque la commande SQL retourne un nom de fichier qui comporte des caractères invalides pour un nom de fichier, ceux-ci sont remplacés par un "_". Par exemple "14:38.txt" deviendra "14_38.txt".
  • Correction d'un bogue qui faisait que la fenêtre principale de l'application gelait si un popup annonçait un échec d'une assertion.

Version 1.0.7

  • Ajout de la version de JCAssertion dans le fichier de journal.
  • Ajout de la date et heure d'exécution de l'essai dans le fichier de journal.
  • Ajout du type d'assertion ChargeLOB
  • Ajout du type d'assertion ExporteLOB
  • Pour AssertSQL lorsque la condition est fausse, la valeur réelle est affichée dans le journal.

Version 1.0.6

  • Ajout de SQLExecute.

Version 1.0.5

  • FichierExiste traite maintenant les répertoires et les fichiers.
  • Ajout des balises ConnectionOracle et AssertSQL
