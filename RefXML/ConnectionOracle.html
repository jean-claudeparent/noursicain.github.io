ConnectionOracle

Description

ConnectionOracle permet de spécifier l'information de base pour ouvrir une connection à une base de données oracle. Cette assertion permet aussi d'ouvrir immédiatement la connection si la base de données sera utilisée beaucoup et de fermer la connection lorsque l'on a terminé. Cette connection est faite en utilisant le data oracle provider pour .net.

Balises

Nom Obligatoire Description
Action Non "Ouvrir" ouvre immédiatement la connection à la base de données et la laissera ouverte jusqu'à une fermeture explicite.
"Fermer" ferme la connection à la base de données.
Si aucune action n'est spécifiée la base de données sera ouverte et refermée à chaque assertion qui utilise la banque comme par exemple AsserSQL.
Cache Non Dès que cette balise est présente et contient au moins un caractère, le user et le password n'apparaîtrons pas dans le journal.
Cela permet entre autres, de transmettre à une autre personne le journal sans que cela diffuse les codes d'utilisateurs  et mots de passe utilisés pour se connecter à la base de données.
Password Oui Spécifie le mot de passe à utiliser pour se connecter à la base de données Oracle.
Serveur Non Spécifie le serveur oule nom de l'instance oracle où l'on doit se connecter. Par défaut c'est l'instance oracle par défaut installée sur l'ordinateur qui exécute JCAssertion.
User Oui Spécifie le cômpte d'utilisateur  à utiliser pour se connecter à la base de données Oracle.

Exemple

 <Assertion>
  <Description>Ouvrir la connection à la base de données. </Description>
  <Type>ConnectionOracle</Type>
  <User>JCA</User>
  <Password>{{motdepasse}}</Password>
  <Serveur>{{mdevserveur}}</Serveur>
  <Action>Ouvrir</Action>
  <Cache>x</Cache>
  </Assertion>
