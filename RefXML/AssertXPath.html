AssertXPath

Description

ContenuFichier permet de vérifier si un fichier texte contient un ou plusieurs texte et si il ne contient pas un ouplusieurs textes. Typiquement on vérifie que les textes des résultats atendus sont dans le fichier et qu'aucun message d'erreur n'est présent.

Balises

Nom Obligatoire Description
Contient Non obligatoire  Ne sélectionne que les noeuds qui ont cette chaîne de caractères  dans leur texte. Si le noeud est lui-même une structure xml tous les textes la composant sont concaténés avant de rechercher la chaîne. La recherche fait une différence enre les majuscules et les minuscules. Par exemple 'test' ne sera pas trouvé dans 'Ceci est un TEST'.
ContientMajus Non obligatoire  Comme 'Contient'. mais ne tiens pas compte de la différence entre les majuscules et les minuscules. Par exemple 'Allo' sera trouvé dans 'allo le monde'.
Expression Obligatoire  Expression XPath qui doit sélectionner certains noeuds du fichier XML à traiter.
Fichier Obligatoire Nom du fichier XML qui sera utilisé pour évaluer l'expression XPath. Comprends le chemin. Le fichier XML doit être bien structuré en xml pour que les fonctions xml du frameword .net puisse le traiter. i le fichier est mal formé l'assertion sera fausse peu importe l'expression XPath. Le fichier ne doit pas être trop gros mais cela dépends des caractéristiques de l'ordinateur qui exécute JCAssertion.
MessageEchec Non obligatoire  Message à véhiculer si l'assertion est fausse. .
Operateur Non obligatoire  Opérateur de comparaison à appliquer sur le résultat réel et le résultat attendu pour évaluer si l'assertion est vraie. Voir les valeurs permises dans le tableau plus bas. Si la balise n'est pas  spécifiée la valeur par défaut est 'PG' (Valeur réelle plus grande que valeur attendue).
ResultatAttendu Non obligatoire  Valeur à comparer avec le résultat réel pour déterminer si l'assertion est vraie. Doit contenir un nombre entier .
