AssertSQL

Description

AssertSQL permet de définir une assertion à tester. L'assertion est composée d'une commande SQL qui retourne une seule colonne et une seule rangée. Cette valeur est comparée à un résultat attendu. Un opérateur de comparaison est permis si la valeur attendue est un nombre. Si la commande SQL ne retourne aucune rangée ou si le résultat est null l'assertion est considérée comme fausse. Une assertion ConnectionOracle doit avoir réussi avant que AssertSQL puisse être utilisé.

Balises

Nom Obligatoire Description
AttenduNombre Obligatoire sauf si AttenduTexte est utilisé. Définit le résultat numérique attendu dans l'assertion.
AttenduTexte Obligatoire sauf si AttenduNombre est utilisé. Définit le résultat texte (chaîne de caractère) attendu dans l'assertion.
AttenduNombre Obligatoire sauf si AttenduTexte est utilisé. Définit le résultat numérique attendu dans l'assertion.
MessageEchec Non. Texte qui explique les conséquences du fait que l'assertion est fausse. Par exemple "L'utilisateur n'a pas été crée dans la table TableUtil dans la base de données".
Operateur Non Définit l'opérateur logique à utiliser pour comparer le résultat de l'énoncé SQL et le résultat attendu. Si non spécifié, "est`égal" est pris comme valeur par défaut. L'opérateur est pris en compte juste pour les résultats attendus qui sont des nombres. Voir le tableau plus bas pour les valeurs valides d'opérateur.
SQL Oui Définit l'énoncé SQL qui donnera le résultat à comparer. L'énoncé SQL doit retourner une seule valeur et une seule rangée. Ne pas mettre de ";" à la fin de la commande SQL. On peut faire des enter dans la commande SQL pour qu'elle soit plus lisible.

Liste des opérateurs:

Code d'opérateur Signification (l'assertion est vraie si)
=  Le résultat de l'énoncé SQL est égal à la valeur attendue.
!= Le résultat de l'énoncé SQL est différent de la valeur attendue.
pg Le résultat de l'énoncé SQL est plus grand que la valeur attendue.
pp Le résultat de l'énoncé SQL est plus petit que la valeur attendue.
pg= Le résultat de l'énoncé SQL est plus grand ou égal à la valeur attendue.
pp= Le résultat de l'énoncé SQL est plus petit ou égal à la valeur attendue.

Exemple

 <Assertion>
  <Description>Vérifier que le solde du client est positif après paiementr. </Description>
  <Type>AssertSQL</Type>
  <SQL>select solde
from TableClient
where NoClint = '00001'</SQL>
  <Operateur>pg=</Operateur>
  <AttenduNombre>0</AttenduNombre>
  <MessageEchec>Apr`s que le paiement soit passé, le solde devrait ne plus être négatif.</MessageEchec>
</Assertion>
