Liste des variables générées autommatiquement

Certaines variables sont définies et initialisées automatiquement par JCAssertion.exe. En voici la liste.

Nom Description
JCA.Expression Quand une assertion AssertSQL est évaluée correctement(c'est à dire sans erreur technique), l'expresssion de comparaison est mise dans cette variable. Cette variable peut donc être utilisée dans le message d'échec de AssertSQL ou dans d'autres assertions qui suivent l'assertion AssertSQL. Le contenu est oujours celui de la derni`re assertion AssertSQL qui a été évaluée. Cette variable n'est pas définie tant que l'on a pas évaluée une assertion AssertSQL.
JCA.FichierDeVariables Si JCAssertion.exe a lu ses variables à partir d'un fichier, le nom de ce fichier, avec son chemin est mis comme valeur de cette variable. La variable n'est pas définie si aucun fichier  de variabkes n'a été lu.
JCA.FichierDeVariables Si JCAssertion.exe a lu ses variables à partir d'un fichier, le nom de ce fichier, avec son chemin est mis comme valeur de cette variable. La variable n'est pas définie si aucun fichier  de variabkes n'a été lu.
JCA.FichierDeVariables Si JCAssertion.exe a lu ses variables à partir d'un fichier, le nom de ce fichier, avec son chemin est mis comme valeur de cette variable. La variable n'est pas définie si aucun fichier  de variabkes n'a été lu.
